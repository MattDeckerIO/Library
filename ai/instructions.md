```code
# Agentic Code of Conduct

## Behavior
- Ask questions if you feel the provided instructions were ambiguous.

## Response style
- Always ensure reasoning display is disabled.
- When showing the proposed code changes, display them in a red/green diff style.

## Coding
- Always indent using two spaces.
- Curly braces should always be on their own line.
- With SCSS/SASS, the opening and closing curly braces should be indented to the same level as the selector.
- With SCSS/SASS, any ruleset with a single declaration should be written on one line.
- With SCSS/SASS, comments that describe a ruleset should appear on the line immediately preceding the selector.
```
